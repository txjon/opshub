Route reference
What each API route does, in plain English.
/api/email/send
Generic transactional send. Accepts type of quote / invoice / invoice_proofs / po. Fetches the matching PDF route, attaches it, sends via Resend, logs to email_messages + job_activity.
/api/email/notify
Auto-notifications triggered by state changes. Handles order_shipped_vendor / order_shipped_hpd / order_shipped / production_complete / invoice_revised directly (with PDF attachments) and delegates everything else to sendClientNotification().
lib/auto-email.ts
sendClientNotification() — fallback for no-attachment notifications. Handles proof_ready / quote_approved / payment_received / tracking_update / invoice_ready / production_complete / order_shipped_*. Includes dedup guard for once-per-job types.
/api/qb/webhook2
Active QuickBooks webhook. On a Payment event, records the payment, notifies the team, and fires an inline Resend send with the PAID-stamped invoice PDF. webhook1 is the older version — still present but superseded.
/api/cron/alerts
Runs on Vercel cron. Scans active jobs for stalled items, overdue payments, upcoming ship dates; emails a digest to OWNER_EMAIL and also writes in-app notifications to team members.
/api/email/compose
Generic body-driven send — takes a pre-built subject/body and sends. Legacy of the removed in-app email composer. No UI references this anymore; route is still mounted.
Design rules currently in code
  • 3-email rule per job: proofs + invoice + shipping (per shipment). Other auto-notifications are throttled by once-per-job dedup.
  • Recipient fallback: billing → primary → any contact with email. Ship-related emails always prefer billing so AP sees them.
  • Per-vendor ship emails: drop-ship jobs fire once per decorator when their slice is shipped. Client sees one email per vendor shipment, not one per item.
  • Invoice revision: pushing updated qtys to QB auto-sends a revised invoice to the client. Triggered from variance review or any re-push.
  • Archive of sends: every outbound email writes a row to email_messages + an auto log to job_activity. Visible on the project activity feed.
