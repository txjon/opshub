System Overview

What is OpsHub?
OpsHub manages every project from the moment a client reaches out to the moment their order ships. It tracks items, pricing, artwork, invoicing, production, and fulfillment — all in one place.

Key concept: Every project flows through stages. The system auto-advances through these stages as you complete actions. You don't manually change phases — the system knows where you are based on what's been done.

Project Workflow — Start to Finish

The 10-Step Flow
  1. Create Project Taylor
    Projects → New Project. Select client (or create new). Set shipping route and ship date.
  2. Add Items Taylor
    Product Builder tab. Drop PSD + mockup files to create items, or use the catalog (S&S, AS Colour, etc.). Enter sizes and quantities.
  3. Cost the Items Drake
    Costing tab. Select decorator, add print locations, set margin. Click "Lock In Pricing" when done.
  4. Send Quote Drake
    Quote tab. Preview the quote, then send via email. Client receives a portal link to review and approve.
  5. Client Approves Quote
    Client clicks "Approve Quote" on their portal. System auto-advances the project.
  6. Send Proofs & Invoice Drake
    Proofs & Invoice tab. Generate proofs, create QB invoice, send both to client.
  7. Client Pays & Approves Proofs
    Client pays via QuickBooks payment link. Approves proofs on portal. Both auto-recorded.
  8. Order Blanks & Send PO Drake
    Blanks tab — enter S&S order numbers. PO tab — send purchase orders to decorators.
  9. Production & Tracking Drake
    Monitor production. Enter tracking when decorator ships. System auto-advances items.
  10. Receive & Ship Receiver
    Receiving page — confirm quantities. Warehouse ships to client. Project complete.
↓
Key rule: The system auto-advances phases. You never manually change a project's phase. Just do your work on each tab, and the system figures out where the project is.

Navigation & Layout

App Layout
Left rail: Hamburger menu (☰) opens the global navigation — Dashboard, Projects, Clients, Decorators, Production, etc. Click any link to navigate. Click outside to close.

Inside a project:
• Top bar: Project header with client name, job number, KPIs (revenue, cost, margin), and ship date countdown
• Progress bar: Horizontal tabs showing workflow stages. Click any tab to navigate. Green = done, amber = in progress, gray = not started.
• Left sidebar: Item list (on Product Builder and Costing tabs). Click an item to view/edit it. Drag to reorder.
• Main content: The active tab's content for the selected item.

Product Builder Taylor

Adding Items
Method 1 — File drop: Drag PSD and mockup PNG files onto the drop zone. Files are matched by name:
• front-tee.psd + front-tee-mockup.png → creates "front-tee" with art + mockup
• Multiple pairs create multiple items in one drop

Method 2 — Catalog: Click "+ Add Item" → choose source (Favorites, S&S, AS Colour, LA Apparel, Other)

Method 3 — Custom Accessory: For non-garment items (patches, stickers, etc.)
Working with Items
Sidebar: Shows all items. Click to select. Drag ⠿ handle to reorder.

Item card: Shows mockup thumbnail, blank info (click to change), size inputs, and files.
• Item name: Editable in the header bar — click to type
• Blank: Click the blank bar to change supplier/style
• Sizes: Enter quantities per size. Tab/arrow keys to navigate between fields.
• Dist button: Auto-distribute a total quantity across sizes using a curve
• Files: Shows uploaded files by stage. Drop more files to add.
Auto-save: All changes save automatically after a short delay. No save button needed. If there's an error, you'll see a red toast at the bottom right.

Costing Drake

How Costing Works
Each item has two panels:

Blanks (left): Supplier selection, fleece toggle, size breakdown with per-size costs, item notes, and financial summary (revenue, blanks cost, PO total, margin).

Decoration (right): Decorator selection, print locations with color counts, tag print, packaging & finishing, setup fees, specialty options, and custom costs.

Margin tiers: Set in the project totals sidebar (10%, 15%, 20%, 25%, 30%). This determines the sell price.

Sell $/unit override: Click "override" next to the sell price to set a custom price instead of the calculated one.
Lock In Pricing
When all items are costed, click "Lock In Pricing" at the top of the costing tab. This:
• Marks pricing as confirmed
• Turns the Costing progress step green
• Unlocks the Quote tab
• Makes costing inputs read-only (prevents accidental changes)

To make changes after locking, click "Unlock Pricing" — this is intentional so changes are deliberate.
Important: The Quote tab won't activate until pricing is locked. This prevents sending a quote with incomplete or changing pricing.

Quote & Client Approval Drake

Sending a Quote
  1. Go to Quote tab
    Preview the quote with all items, sizes, and pricing. Download PDF to review if needed.
  2. Send to client
    Click "Send Quote" — email dialog opens. Select recipient. Quote PDF is attached automatically.
  3. Client reviews on portal
    Client receives email with portal link. They see the quote with full detail — items, sizes, pricing, totals.
  4. Client approves or requests changes
    Approve → project advances. Request Changes → you'll see a red alert on the dashboard with their notes.
Quote sent log: After sending, the quote tab shows "Sent — Quote emailed [date] at [time]" so you know the current state.

Ordering Blanks Drake

3-Gate Checklist
Before you can order blanks, three gates must be met:

✓ Quote approved — client has approved the quote
✓ Payment received — depends on terms (net terms auto-pass)
✓ All proofs approved — client has approved all proofs

All three show as green checkmarks at the top of the Blanks tab. Once met, enter the S&S order number and cost for each item.

Purchase Orders Drake

Sending POs to Decorators
  1. Go to PO tab
    PO renders with all items for each decorator, with decoration details and pricing.
  2. Fill per-item fields
    Production files link, incoming goods notes, production notes, packing notes. Use "Copy to all" for common values.
  3. Preview PO PDF
    Check bill-to, ship-to, shipping account, and all item details.
  4. Send PO via email
    Select decorator contacts. PO PDF attaches automatically.
Shipping account: UPS orders automatically get shipping account W28Y51. Ship-to defaults to HPD warehouse (4670 W Silverado Ranch) for ship-through/stage routes.

Production Tracking Drake

How Items Move Through Production
After PO is sent: Items are "at decorator" — visible on the Production page.

When decorator ships: Enter tracking number on the Production tab. Item auto-advances to "shipped."

Dashboard alerts:
• 3-6 days at decorator: Purple "watch" alert
• 7+ days at decorator: Amber "stalled" alert
• Items shipped: Green "incoming to warehouse" alert
• 10+ days in transit: Amber "late receiving" alert

Warehouse & Shipping Receiver

Receiving Items
Receiving page shows all items shipped from decorators that haven't been confirmed at HPD.

For each item:
• Confirm received quantities per size (defaults to shipped qty)
• Flag any variances (short shipment, damage)
• Click "Confirm Received"

After all items received:
• Ship-through: Enter outbound tracking → ship to client → complete
• Stage: Goes to fulfillment → pack → ship → complete
• Drop-ship: Skips warehouse entirely — complete when tracking entered

Dashboard Command Center Everyone

Two Columns — Sales & Production
The dashboard shows every action needed across all active projects. Each alert is a clear command.

Sales column: Quote sending, invoice creation, proof uploads, payment follow-ups, overdue projects, client rejections

Production column: Order blanks, send POs, stalled items, incoming shipments, fulfillment, shipping deadlines

Click any alert to take action — opens the right tab or modal. Alerts disappear automatically when the action is completed.

2-Day Follow-Up Rule
When the system is waiting on a client (quote sent, proofs sent, invoice sent), alerts automatically escalate after 2 days with no response:

Day 0-2: No alert (give client time)
Day 2+: Amber "Follow up — sent Xd ago, no response"

This ensures no client communication falls through the cracks.

Client & Vendor Portals

Client Portal
Each project has a unique portal link (copy from project header). Clients use it to:
• View their quote with full detail (sizes, pricing, totals)
• Approve or reject quotes (with notes)
• Review and approve proofs in a fullscreen viewer
• Pay via QuickBooks payment link
• View invoice PDF
• See project updates and progress

Clients with multiple projects see all of them in a sidebar — they can switch between projects without separate links.
Vendor Portal
Each decorator has a unique portal link (from Decorators page). They use it to:
• View all active POs with full detail
• See item details, decoration specs, quantities, and ship-to address
• Confirm PO receipt
• Report issues
• Enter tracking when items ship

Active orders in a sidebar list, sorted by ship date. Completed orders in a separate tab with search.

QuickBooks Integration

Invoice + Payment Flow
Creating an invoice:
• Click "Create QuickBooks Invoice" on the Proofs & Invoice tab (or from Dashboard)
• Or enter a manual invoice number if the invoice already exists in QB

Payment:
• Client pays via the QB payment link (included in the invoice email)
• Payment is automatically recorded in OpsHub via webhook
• Shows in the project, on the dashboard, and in the client portal

If a payment doesn't register: Use the manual sync tool (ask Jon)
