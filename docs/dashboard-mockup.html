Design Notes
Every alert is a unique command
No shared pill labels. Each card's action text IS the message — reads like an email subject line commanding an action. The left color border indicates priority tier, the text tells you exactly what to do.
Rejections surface client notes
Quote rejections and proof revisions show the client's actual notes in a quoted block. Clicking navigates to where the correction can be reviewed and applied:
• Quote rejected → Quote tab (costing adjustments)
• Proof revision → Art tab (file re-upload)
Priority tiers by color
Red — Critical: overdue, rejections with client notes (sort to top)
Amber — High: create invoice, send invoice (prepaid, gates production), stalled at decorator
Purple — Medium: send quote, send invoice (net, doesn't gate), upload proofs
Muted — Informational: awaiting approval (no action needed, just awareness)
Blue — Production: order blanks, send PO
Green — Incoming: items shipped to warehouse
Click actions
• Send quote → Email modal (or no-contacts popup if no email)
• Quote rejected → Quote tab — see notes, adjust costing, re-send
• Create invoice → Invoice modal (manual # or push to QB)
• Send invoice → Email modal (or no-contacts popup)
• Upload proofs → Art tab
• Awaiting approval → Approvals tab (informational)
• Proof revision → Art tab — see notes, revise, re-upload
• Overdue → Project overview
• Order blanks → Blanks tab
• Send PO → PO tab
• Stalled → Production tab
• Incoming → Warehouse
New: Quote rejection (requires portal update)
Client portal currently only has "Approve Quote." Needs a "Request Changes" button with a notes textarea. Saves to jobs.quote_rejection_notes + sets jobs.quote_rejected = true. Dashboard alert pulls the notes and displays them inline. Clicking takes Drake to the quote tab to adjust pricing/items, then re-send.
Invoice # already exists
If a job already has an invoice number in type_meta when the quote is approved, the "Create invoice" alert never fires — flow skips straight to "Send invoice." This handles the case where an invoice was pre-created in QuickBooks.
