Phase Pills

Shown on job detail header, projects list, and dashboard. These use a mix of T.xxx and hardcoded hex — inconsistencies flagged below.

intake pending ready production receiving shipping fulfillment complete on_hold cancelled
Inconsistency: Phase colors use hardcoded hex (#EEEDFE, #3C3489, #E6F1FB, #0C447C, etc.) instead of the T.xxx theme system. "pending" and "production" use custom blue/indigo not in the theme. "receiving" uses custom warm tones not in the theme. Only "ready", "fulfillment", "on_hold" use T.xxx colors.

Payment Status

draft sent viewed partial paid overdue void
Inconsistency: draft, sent, viewed, and partial all use T.amber — no visual distinction between these states. A client seeing "sent" vs "viewed" vs "partial" all look identical.

Priority Flags

Hot Rush Normal
Inconsistency: Priority colors use dark backgrounds with light text — a holdover from the old dark theme. These don't match the light-theme pill pattern used everywhere else (light bg, dark text).

Proof Approval

Approve Approved Revise Generate Proof
Inconsistency: "Revise" and "Generate Proof" are solid-fill buttons while "Approve/Approved" are outline-style pills. Mixed border-radius (6px vs 99px used elsewhere).

Decoration & Item Type Badges

screen print embroidery Accessory
Inconsistency: Decoration types use borderRadius: 6, while status pills use borderRadius: 99. "Accessory" uses purple while decoration types use accent gray.
